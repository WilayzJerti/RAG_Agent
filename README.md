# RAG Agent

Проект для автоматизированной обработки вопросов и генерации ответов с использованием технологии Retrieval-Augmented Generation (RAG) на базе локальной LLM модели.

## Описание проекта

RAG Agent использует локальную языковую модель (LLM) `gemma4` через Ollama для генерации ответов на вопросы на основе предоставленной документации. Проект позволяет массово обрабатывать таблицы с вопросами и сравнивать полученные ответы с эталонными.

## Требования

- Python 3.8 и выше
- Ollama (для запуска локальной LLM)
- 4+ ГБ оперативной памяти
- 8+ ГБ свободного места на диске (для модели gemma4)

## Установка и настройка

### 1. Создание виртуального окружения Python

```bash
python -m venv venv
```

### 2. Активация виртуального окружения

**На Windows:**
```bash
venv\Scripts\activate
```

**На macOS/Linux:**
```bash
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Установка и настройка Ollama

#### Скачивание Ollama

Скачайте Ollama с официального сайта: https://ollama.ai

#### Установка модели gemma4

После установки Ollama выполните команду:

```bash
ollama pull gemma4
```

#### Запуск Ollama

```bash
ollama serve
```

По умолчанию Ollama запустится на `http://localhost:11434`

## Подготовка файлов данных

Разместите следующие файлы в корневой папке проекта (в одной директории со скриптом `rag_agent.py`):

### 1. `document.txt`
- Файл с исходной документацией или информацией для LLM
- Может содержать текст из Google Docs или другого источника
- Используется для контекста при генерации ответов

Пример структуры:
```
[Содержимое документации, которое будет использоваться для генерации ответов]
```

### 2. `RAG_questions_table.csv`
- CSV таблица с вопросами для обработки
- Должна содержать колонку с вопросами (например, `question` или `Question`)

Пример структуры:
```csv
question,category
"Как создать виртуальное окружение?","setup"
"Что такое RAG?","concepts"
"Как установить зависимости?","setup"
```

### 3. `RAG_answers.csv`
- CSV таблица с эталонными (правильными) ответами на вопросы
- Должна содержать колонку с ответами для сравнения

Пример структуры:
```csv
answer,question_id
"Используйте команду python -m venv venv",1
"RAG - это технология для генерации ответов с использованием информации из документов",2
"Используйте pip install -r requirements.txt",3
```

## Запуск проекта

### 1. Убедитесь, что Ollama запущена

```bash
# Проверить статус (в отдельном терминале)
curl http://localhost:11434
```

### 2. Запустите скрипт RAG Agent

```bash
python rag_agent.py
```

### 3. Ожидание результатов

Скрипт будет обрабатывать вопросы из `RAG_questions_table.csv` по одному. Это может занять некоторое время в зависимости от:
- Количества вопросов
- Размера документации
- Мощности вашего компьютера

### 4. Получение результатов

После завершения работы скрипта будут созданы следующие файлы:

#### `my_answers.csv`
- CSV файл с полученными ответами LLM
- Содержит сопоставление между вопросами и сгенерированными ответами

Пример структуры:
```csv
question,generated_answer,reference_answer,similarity_score
"Как создать виртуальное окружение?","Используйте команду python -m venv","Используйте команду python -m venv venv",0.95
```

## Решение проблем

### Ollama не запускается

```bash
# Проверьте порт 11434
# На Windows:
netstat -ano | findstr :11434

# На macOS/Linux:
lsof -i :11434
```

### Модель gemma4 не загружается

```bash
# Повторите попытку загрузки
ollama pull gemma4

# Проверьте доступный объем памяти и места на диске
```

### Скрипт не находит файлы данных

- Убедитесь, что файлы находятся в корневой папке проекта
- Проверьте точные названия файлов (чувствительны к регистру)
- Используйте полные пути, если файлы в других директориях

### Низкая скорость обработки

- Это нормально для локального запуска LLM
- Рассмотрите использование более слабой модели или сокращение размера документации
- Убедитесь, что нет других ресурсоемких приложений

## Дополнительная информация

### Что такое RAG?

RAG (Retrieval-Augmented Generation) - это подход, который комбинирует:
1. **Retrieval (Поиск)** - поиск релевантной информации в документах
2. **Augmentation (Дополнение)** - добавление найденной информации в контекст запроса
3. **Generation (Генерация)** - использование LLM для создания ответа на основе документа и запроса

### Преимущества локального запуска LLM

- 🔒 Приватность - данные не отправляются на внешние серверы
- 💰 Экономия - нет затрат на API вызовы
- 🚀 Скорость - после загрузки модели работает без интернета
- ⚙️ Контроль - полный контроль над моделью и процессом
